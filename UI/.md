# Interactive-Assets Project Memory

## Project Overview
Interactive-Assets is a pnpm workspace monorepo containing an educational search engine visualization application called "QueryFlow". The project demonstrates how search engines crawl, process, index, and rank information through interactive visualizations.

## Monorepo Structure

### Libraries (lib/)
- **api-spec**: OpenAPI 3.1.0 specification for the API
  - Contains `openapi.yaml` with health check endpoint
  - Uses Orval to generate React Query client and Zod schemas
  - Title must remain "Api" to avoid breaking import paths

- **api-client-react**: Generated React Query client
  - Auto-generated from OpenAPI spec using Orval
  - Uses custom fetch mutator
  - Exports generated hooks for API calls

- **api-zod**: Generated Zod validation schemas
  - Auto-generated from OpenAPI spec using Orval
  - Includes TypeScript types
  - Supports date and bigint coercion

- **db**: Database layer using Drizzle ORM
  - PostgreSQL database with Drizzle ORM
  - Schema is currently empty (no tables defined yet)
  - Uses `drizzle-zod` for schema validation
  - Exports `db` instance and `pool` for connections
  - Commands: `pnpm run push` (dev), `pnpm run push-force`

### Artifacts (artifacts/)

- **api-server**: Express 5 API server
  - Port configured via PORT environment variable
  - Uses Pino for logging
  - CORS enabled
  - Routes mounted at `/api`
  - Currently has health check endpoint only
  - Build with esbuild (CJS bundle)
  - Commands: `pnpm run dev`, `pnpm run build`, `pnpm run start`

- **ir-search**: Main React application - QueryFlow
  - Educational search engine visualization tool
  - Built with React 19, Vite, TypeScript
  - Uses Wouter for routing
  - TanStack Query for data fetching
  - TailwindCSS + Radix UI components
  - Framer Motion for animations
  - Pages:
    - `/`: Home page with search bar and feature cards
    - `/results`: Search results page
    - `/journey`: Educational journey explaining search engine concepts
  - Features:
    - Inverted Index visualization
    - TF-IDF Ranking explanation
    - Real-time search with mock corpus
    - Search suggestions
  - Authors: Mostafa Zahran and Saeed Mohamed
  - Commands: `pnpm run dev`, `pnpm run build`, `pnpm run serve`

- **mockup-sandbox**: Component preview server
  - Renders individual components for workspace canvas
  - Dynamic component loading from `src/components/mockups/`
  - Access previews at `/preview/ComponentName`
  - Uses custom Vite plugin for component discovery
  - Commands: `pnpm run dev`, `pnpm run build`, `pnpm run preview`

### Scripts (scripts/)
- Utility scripts workspace
- Contains post-merge hook that runs `pnpm install` and `pnpm --filter db push`

## Application Purpose

QueryFlow is an educational tool designed to help users understand how search engines work internally. It provides:
- Visual explanations of the search pipeline (crawling, processing, indexing, ranking)
- Interactive demonstrations of inverted indexes
- TF-IDF ranking algorithm visualization
- Real-time search experience with a mock document corpus
- Step-by-step journey through the data processing pipeline

The target audience appears to be students or developers interested in Information Retrieval (IR), Natural Language Processing (NLP), and Data Mining concepts.

## Search Engine Implementation

### Mock Data Structure
**Location:** `src/lib/mock-data.ts`

**Document Interface:**
```typescript
interface Document {
  id: string;
  title: string;
  url: string;
  snippet: string;
  tags: string[];
}
```

**Mock Corpus:** 12 documents covering IR, NLP, search engines, crawling, indexing, ranking algorithms

### Search Engine Logic
**Location:** `src/lib/search-engine.ts`

**Features:**
1. **Tokenization**: Lowercase, remove punctuation, split on whitespace, filter stop words
2. **Inverted Index**: Maps terms to document IDs with term frequencies
3. **Document Frequencies**: Tracks how many documents contain each term
4. **TF-IDF Ranking**: 
   - Normalized term frequency (TF)
   - Inverse document frequency (IDF)
   - Score = normalized TF * IDF
5. **Search Suggestions**: Simple substring matching on document titles (top 5)

**Stop Words:** Common English words (a, an, and, are, as, at, be, but, by, for, if, in, into, is, it, no, not, of, on, or, such, that, the, their, then, there, these, they, this, to, was, will, with, how, what, where, why, when, who)

### Database Schema

**Table: documents**
```sql
CREATE TABLE documents (
  id SERIAL PRIMARY KEY,
  title TEXT NOT NULL,
  url TEXT NOT NULL,
  snippet TEXT NOT NULL,
  tags TEXT[] NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- Index for faster search
CREATE INDEX idx_documents_title ON documents USING gin(to_tsvector('english', title));
CREATE INDEX idx_documents_tags ON documents USING gin(tags);
```

**Drizzle Schema:**
```typescript
import { pgTable, serial, text, timestamp, array } from 'drizzle-orm/pg-core';

export const documents = pgTable('documents', {
  id: serial('id').primaryKey(),
  title: text('title').notNull(),
  url: text('url').notNull(),
  snippet: text('snippet').notNull(),
  tags: array(text('tags')).notNull(),
  createdAt: timestamp('created_at').defaultNow(),
  updatedAt: timestamp('updated_at').defaultNow(),
});
```
